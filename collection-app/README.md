# אפליקציית ניהול גבייה (סטטית, GitHub Pages)

אפליקציית ווב חד-קובצית (`index.html`) שעובדת מכל מכשיר/דפדפן עם קישור אחד — בדיוק כמו אפליקציית הטיסות (Ufly-Flights). אין שרת מאחורי הקלעים: האפליקציה מתחברת ישירות מהדפדפן לחשבון Google (OAuth) ול-Google Sheets.

## תכונות
- ניהול לקוחות נפרדים, כל אחד עם קישור משלו ל-Google Sheet
- רשימת גבייה לכל לקוח עם הוספה/עריכה/מחיקה
- סנכרון הדדי אוטומטי מול Google Sheets (כל דקה + סנכרון ידני)
- תזכורות לפי "תאריך מעקב"
- שליחת תזכורת ללקוח בקליק - פותח WhatsApp או מייל עם הודעה מוכנה
- שיתוף קישור לאפליקציה (כולל קוד QR) כדי לפתוח אותה מכל מכשיר

## הפעלה ב-GitHub Pages
1. ב-Settings -> Pages של ה-repo, יש לבחור את הענף (branch) ואת התיקייה `collection-app` כמקור
2. הכתובת שתתקבל (למשל `https://nunah31.github.io/Lea-Collection/collection-app/`) היא הקישור הקבוע לאפליקציה - אפשר לשלוח אותו לכל מכשיר

## התחברות ל-Google (חד פעמי לכל מכשיר/דפדפן)
בלחיצה על "חיבור ל-Google Sheets" באפליקציה מופיעות הוראות להגדרת **OAuth Client ID** מסוג **Web application** ב-Google Cloud Console, עם "Authorized JavaScript origins" שמצביע לכתובת בה האפליקציה מתארחת. לאחר הגדרה חד-פעמית, כל מכשיר ילחץ "התחברות לחשבון Google" ויאשר גישה (חיבור נשמר בדפדפן עד שפג תוקפו, כשעה - אפשר להתחבר מחדש בקלות).

## חשוב לדעת
- רשימת הלקוחות וההגדרות (כולל ה-Client ID) נשמרות **בדפדפן/מכשיר** (localStorage), כמו ביתר האפליקציות (Ufly-Flights). מכשיר חדש יצטרך הגדרה ראשונית קצרה (הוספת לקוחות + חיבור לגוגל).
- נתוני הגבייה עצמם מסונכרנים ל-Google Sheets המשותף - כך שכל מכשיר רואה את אותם נתונים אחרי סנכרון.
- העמודות בקובץ ה-Sheet חייבות להתאים לתבנית: משימה | תאריך מעקב | שם לקוח | נייד לקוח | כתובת מייל | סכום החוב | מס' חשבונית | יתרה לתשלום נכון ליום | אמצעי תשלום | הערות
